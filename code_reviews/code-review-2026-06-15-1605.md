# Code Review — 2026-06-15 16:05

## Findings

### [P1] A partially indexed boundary line is read through EOF

`byteRangeForLine` treats any indexed line without a following index entry as the physical last line and returns `start..<fileSize`. The app indexes only through `endLine` and then reads that line, so line 21 of a file with more than 21 lines includes every remaining byte in the file. On a multi-GB input this can allocate and tokenize nearly the entire file during initial open, while also marking the combined text as invalid JSON.

- [Sources/BigJSONLCore/Indexing/LineOffsetIndex.swift:120](../Sources/BigJSONLCore/Indexing/LineOffsetIndex.swift#L120)
- [Sources/BigJSONLApp/ViewModel/DocumentViewModel.swift:83](../Sources/BigJSONLApp/ViewModel/DocumentViewModel.swift#L83)

The index needs to distinguish “known EOF” from “next line not indexed yet,” or callers must index one line beyond every line whose range will be requested.

#### Resolution — Fixed

`LineOffsetIndex` now tracks scan progress and confirmed EOF, and returns no range until a next-line offset or EOF is known. App and CLI callers index one lookahead line. `A line range requires a known next boundary or EOF`, the app viewport test, and the 50-line CLI check all passed; the CLI rendered exactly lines 1–20 without consuming line 21 or later.

### [P1] The GUI cannot scroll beyond its initial finite window

`ContentView` renders only `visibleLines`, while `loadWindow` is called only during open and explicit `scrollTo`. There is no scroll-position or edge-appearance callback that requests the next window as the user reaches either end. A normal open therefore exposes only the first 21 lines; search can replace that buffer with a different window, but continuous file browsing is not implemented.

- [Sources/BigJSONLApp/Views/ContentView.swift:18](../Sources/BigJSONLApp/Views/ContentView.swift#L18)
- [Sources/BigJSONLApp/ViewModel/DocumentViewModel.swift:61](../Sources/BigJSONLApp/ViewModel/DocumentViewModel.swift#L61)

Add scroll-driven window loading with stable line IDs and preserve the selected/visible anchor when replacing the buffer.

#### Resolution — Fixed

The app now uses top and bottom edge loaders to shift a bounded overlapping viewport, assigns stable line IDs, and restores a shared anchor after each shift. `Viewport advances and retreats with bounded overlapping windows` passed, verifying ranges 1–21, 21–61, and back to 1–21.

### [P1] Search blocks the main actor and buffers every match in memory

`DocumentViewModel.performSearch` synchronously launches and waits for the subprocess on the main actor. `Searcher` simultaneously reads all stdout into one `Data`, converts it to one `String`, splits it, builds every `SearchResult`, and sorts the complete result set. A broad query over the large files this application targets can freeze the UI for the duration and consume memory proportional to total matching output.

- [Sources/BigJSONLApp/ViewModel/DocumentViewModel.swift:124](../Sources/BigJSONLApp/ViewModel/DocumentViewModel.swift#L124)
- [Sources/BigJSONLCore/Search/Searcher.swift:104](../Sources/BigJSONLCore/Search/Searcher.swift#L104)

Run search off the main actor, support cancellation, parse output incrementally, and impose a result/page limit when the caller only needs a viewport or first match.

#### Resolution — Fixed

Search now has an async API that runs outside the main actor, terminates its subprocess on task cancellation, and passes a native maximum-match argument to grep/ripgrep so captured output is bounded. The app retains at most 500 results and the CLI requests one. `Async search responds to task cancellation` and `App search runs asynchronously and caps retained results` both passed.

### [P1] A normal no-match search is reported as a failure

Both ripgrep and grep return status 1 when no lines match. `runSubprocess` accepts only status 0, so `Searcher.search` throws instead of returning an empty result list. This makes the CLI's “No matches found” branch unreachable and causes the app to display a search error for an ordinary outcome.

- [Sources/BigJSONLCore/Search/Searcher.swift:141](../Sources/BigJSONLCore/Search/Searcher.swift#L141)
- [Sources/bigjsonl-cli/BigJSONL.swift:62](../Sources/bigjsonl-cli/BigJSONL.swift#L62)

Treat status 1 as an empty successful result for both supported tools; reserve errors for launch failures and statuses 2 or greater.

#### Resolution — Fixed

Subprocess status 1 now returns an empty result. `No-match searches return an empty result` passed, and a CLI no-match check printed `No matches found.` with exit status 0.

### [P1] The test suite depends on ignored, machine-local fixtures

`testJSONLFile` requires a file under `test-files/`, but the entire directory and all `*.jsonl` files are ignored and none are tracked. The suite passes on this workstation only because local files happen to exist; a clean clone or CI runner will throw while discovering fixtures, so the core indexing and mapped-file tests are not reproducible.

- [Tests/BigJSONLCoreTests/TestHelpers.swift:35](../Tests/BigJSONLCoreTests/TestHelpers.swift#L35)
- [.gitignore:8](../.gitignore#L8)

Generate deterministic temporary JSONL fixtures inside the tests, or explicitly unignore and commit a small synthetic fixture that cannot contain user data.

#### Resolution — Fixed

Core and app tests now generate isolated temporary JSONL files and remove them on teardown; no test reads `test-files/`. The complete 30-test suite passed without relying on ignored repository data.

### [P2] Empty files are indexed and rendered as one invalid line

`ensureLineIndexed` unconditionally appends line 1 before checking `mappedFile.size`. For a zero-byte file, the CLI consequently renders an empty invalid row and reports one line instead of showing an empty document.

- [Sources/BigJSONLCore/Indexing/LineOffsetIndex.swift:64](../Sources/BigJSONLCore/Indexing/LineOffsetIndex.swift#L64)

Return without adding an entry when the mapped file is empty, and cover the behavior with an index-level test.

#### Resolution — Fixed

Empty mappings now mark the index complete without adding line 1. `Empty files contain zero indexed lines` passed, and the CLI check printed `No lines | 0 B` without rendering an invalid row.

### [P2] Invalid viewport sizes can abort the CLI

`windowLines` is a signed `Int` with no validation. A negative value traps when converted to `UInt64`, and zero underflows in the footer expression `startLine + UInt64(windowLines) - 1`, producing nonsensical output today and remaining vulnerable to checked-arithmetic changes.

- [Sources/bigjsonl-cli/BigJSONL.swift:40](../Sources/bigjsonl-cli/BigJSONL.swift#L40)
- [Sources/bigjsonl-cli/BigJSONL.swift:81](../Sources/bigjsonl-cli/BigJSONL.swift#L81)

Validate `windowLines > 0` through ArgumentParser before rendering. The same validation should reject line number 0.

#### Resolution — Fixed

ArgumentParser validation now rejects nonpositive viewport sizes and line 0. Focused checks for `--window-lines=-1`, `--window-lines 0`, and `--line 0` all exited with status 64 and a validation message instead of trapping.

### [P2] `MappedFile.read` can return data that outlives its mapping

The no-copy `DispatchData` uses an empty deallocator and does not retain its owning `MappedFile`. Because `read` is public, a caller can keep the returned data after releasing the file, leaving the data backed by unmapped memory and making later access undefined.

- [Sources/BigJSONLCore/FileIO/MappedFile.swift:58](../Sources/BigJSONLCore/FileIO/MappedFile.swift#L58)
- [Sources/BigJSONLCore/FileIO/MappedFile.swift:83](../Sources/BigJSONLCore/FileIO/MappedFile.swift#L83)

Return an owner-retaining region type, or arrange for the `DispatchData` deallocator closure to retain the mapping owner for the data's lifetime.

#### Resolution — Fixed

The no-copy `DispatchData` deallocator closure now retains its `MappedFile` owner, and range validation avoids unsigned addition overflow. `Mapped data retains its mapping owner` passed, confirming the owner remains alive after the caller releases its direct reference while mapped data is retained.
